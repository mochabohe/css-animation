# CSS Motion — Design Context

## Design Context

### Users

前端开发者和 UI/UX 设计师各半。开发者来这里**复制代码、集成到项目**，需要快速找到目标动画、看到效果、拿走代码；设计师来这里**寻找灵感、理解动效规范**，关注视觉呈现和分类体系。两类用户都要求高效率，不喜欢被界面本身打扰。

### Brand Personality

**三个词：克制、精准、有质感**

这是一个给专业人士用的工具，不是娱乐产品。界面应该像 Linear 或 Vercel — 你几乎感觉不到界面的存在，注意力完全在内容（动画效果）上。

### Aesthetic Direction

**参考方向：** Vercel / Linear / Raycast — 深色、高密度信息、精准的细节处理
**主色：** 青绿调 `#2cb9c5`，冷峻、科技感，区别于主流的蓝色系
**背景：** 深海蓝 `#0b1120`，不是纯黑，有层次感
**反面案例（必须避免）：**
- ❌ 过多装饰性动效（界面动画不能抢动画演示卡片的戏）
- ❌ Bootstrap 式通用感（圆角过大、配色过于柔和）
- ❌ 过于商务 SaaS 的正式感

### Design Principles

1. **动画是主角，界面是舞台** — 所有 UI 选择都服务于让动画卡片更突出。减少界面本身的视觉重量。
2. **克制胜于装饰** — 当你想加一个效果时，先问"去掉它会更好吗"。多数时候答案是肯定的。
3. **密度 × 清晰 = 专业感** — 信息可以密集，但层级必须清晰。开发者要能在 3 秒内完成"找到→预览→复制"。
4. **暗色质感不是暗色氛围** — 深色背景要有层次，用微妙的边框和透明度区分层级，不要一片漆黑。
5. **排版即秩序** — 字号、字重、字色三者配合建立视觉层级，不依赖颜色来传递重要性。

### Technical Context

- **Tech Stack：** Vite + 原生 HTML/CSS/JS（非 React 渲染层）
- **CSS 变量体系：** 完整的 token 系统，accent 色、motion 时长、缓动函数均已参数化
- **主题切换：** 支持暗/亮双主题，`data-color-mode` 属性控制
- **动画速度：** 全局 `--speed-multiplier` 控制，所有时长自动缩放
- **动画分类：** loading / interactive / empty / text / background / wave / motion / bigdata
