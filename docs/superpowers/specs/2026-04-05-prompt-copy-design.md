# 提示词复制功能设计规格

> 为 CSS Motion 动效组件库新增"复制提示词"功能，让用户可以将动画的完整提示词喂给 AI 工具（Cursor/Claude Code/ChatGPT 等），直接在自己的项目中生成对应动画，降低使用门槛。

## 1. 背景与动机

当前用户使用动画的流程是：预览 → 复制 CSS/HTML 代码 → 手动粘贴到项目中适配。这在 vibe coding 时代有使用门槛——用户需要自己找到正确的插入位置并处理技术栈适配（React/Vue/Tailwind 等）。

提示词功能的价值：用户复制一段提示词，粘贴给 AI 编程工具或对话 AI，AI 自动在用户的项目上下文中生成并适配代码。**零门槛、零适配成本。**

## 2. 目标用户与使用场景

- **AI 编程工具用户**：在 Cursor、Claude Code、Copilot 中，让 AI 直接在项目中生成对应动画代码
- **通用 AI 对话用户**：在 ChatGPT、Claude.ai 中获取完整可用的 CSS 动画代码
- 提示词设计为通用格式，两种场景都能用

## 3. 核心方案：描述预存 + 代码动态拼装

### 3.1 提示词由 4 个区组成

| 区块 | 来源 | 内容 |
|------|------|------|
| ① 任务指令 | 固定模板 | 告诉 AI "在我的项目中实现以下 CSS 动画" |
| ② 效果描述 | data.js 预存 | 自然语言描述动画原理、视觉效果、适用场景 |
| ③ 参考代码 | 运行时实时获取 | 当前 HTML + CSS 代码（含用户参数调整后的版本） |
| ④ 使用指引 | 固定模板 | 指导 AI 根据用户项目技术栈适配代码 |

### 3.2 提示词示例

```
请在我的项目中实现以下 CSS 动画效果。

## 效果描述
【原始思维】— 基于玫瑰曲线(rhodonea)数学公式的加载动画。6条SVG路径沿极坐标方程 r=cos(kθ) 绘制花瓣形状，整体匀速旋转，各路径透明度交替变化形成呼吸感。适用于需要科技感的等待场景。
技术要点：SVG, 数学曲线, 旋转, 透明度

## 参考实现

### HTML
​```html
<div class="rose-curve-wrap">
  <svg viewBox="-110 -110 220 220">
    <path d="..." class="petal petal-1" />
    ...
  </svg>
</div>
​```

### CSS
​```css
.rose-curve-wrap { ... }
@keyframes rose-spin { from { transform: rotate(0deg); } to { transform: rotate(360deg); } }
​```

## 使用要求
- 请根据我的项目技术栈（React/Vue/Svelte/原生等）适配代码结构
- 保持动画视觉效果与参考代码一致
- CSS 类名可根据项目命名规范调整
- 如使用 CSS-in-JS、Tailwind 或 CSS Modules，请做相应转换
- 确保动画性能良好，优先使用 transform 和 opacity
```

### 3.3 为什么是这个方案

备选方案：
- **纯静态预存**：提示词中的代码不跟参数调整同步，用户改了时长后提示词还是旧的 — 排除
- **纯模板拼装**：描述部分千篇一律，无法体现每个动画的独特之处 — 排除
- **混合方案（当前）**：描述精准 + 代码实时 + 维护量适中 — 采用

## 4. 数据层设计

### 4.1 新增数据结构

在 `src/data.js` 中新增 `animationPromptDescriptions` 对象：

```js
export const animationPromptDescriptions = {
  "原始思维": {
    description: "基于玫瑰曲线(rhodonea)数学公式的加载动画。6条SVG路径沿极坐标方程 r=cos(kθ) 绘制花瓣形状，整体匀速旋转，各路径透明度交替变化形成呼吸感。适用于需要科技感的等待场景。",
    tags: ["SVG", "数学曲线", "旋转", "透明度"]
  },
  "跳动圆点": {
    description: "3个圆点依次上下弹跳的经典加载动画。使用 translateY + animation-delay 错开节奏，缓动函数为 ease-in-out 制造弹性感。",
    tags: ["弹跳", "延迟序列", "translateY"]
  },
  // ... 105 个动画，AI 批量生成初稿 + 人工校对
};
```

### 4.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `description` | string | 自然语言描述，2-3 句话，涵盖动画原理、视觉效果、适用场景 |
| `tags` | string[] | 关键 CSS/技术标签，帮助 AI 理解核心实现技术 |

### 4.3 数据生成计划

1. 用 AI 基于每个动画的 CSS 代码和名称批量生成 description + tags 初稿
2. 人工校对，确保描述准确反映动画特色
3. 存入 data.js，随代码一起 git 管理

## 5. 提示词组装模块

### 5.1 新增文件 `src/promptBuilder.js`

单一职责：接收动画信息，输出完整提示词字符串。

```
buildPrompt(title, description, tags, htmlCode, cssCode) → string
```

### 5.2 组装逻辑

1. 拼接任务指令（固定文本）
2. 拼接效果描述 + tags（来自 data.js）
3. 拼接参考代码（来自运行时编辑器内容）
4. 拼接使用指引（固定文本）

代码中的 CSS 会经过 `stripPreviewScaleBlock()` 清理预览缩放注释，与现有复制逻辑一致。

## 6. UI 设计

### 6.1 卡片层 — 快捷复制按钮

在每张卡片的按钮区域，与现有"代码实验室"按钮并列新增"复制提示词"按钮：

```
┌──────────────────────────────┐
│ ● 加载反馈                    │
│   原始思维                    │
│     [动画预览区域]            │
│  场景: 页面加载               │
│  ┌──────────┐ ┌──────────┐  │
│  │代码实验室 │ │复制提示词 │  │
│  └──────────┘ └──────────┘  │
└──────────────────────────────┘
```

**交互：**
- 点击 → 组装提示词 → 复制到剪贴板
- 按钮文案变为 "✓ 已复制"，1.5s 后恢复
- 使用卡片当前的原始 HTML + CSS 代码（非参数调整后的版本，因为卡片层没有参数面板）

### 6.2 弹窗层 — 提示词 Tab + 复制按钮

**按钮行：** 新增"复制提示词"按钮，与现有复制按钮并列。

**右侧面板：** 原来只有"CSS 样式"编辑器，改为可切换的两个 Tab：

```
┌───────────┬──────────┐
│ 🎨 CSS样式 │ 📋 提示词 │   ← Tab 切换
├───────────┴──────────┤
│                      │
│  [当前 Tab 的内容]    │
│                      │
└──────────────────────┘
```

**提示词 Tab 内容：**
- 只读文本区域，展示组装后的完整提示词
- 用户在参数面板调整参数 → CSS 编辑器更新 → 提示词中的代码部分同步更新
- 底部有独立的"复制"按钮

### 6.3 样式要求

- 提示词 Tab 的文本区域使用与 CSS 编辑器一致的等宽字体和深色背景
- "复制提示词"按钮使用区分于代码复制按钮的次要样式（不争抢视觉焦点）
- 卡片层按钮尺寸与"代码实验室"一致
- 遵循项目设计原则：克制、不装饰、暗色质感

## 8. 不做的事

- **不做提示词编辑** — 提示词只读，用户复制后可在 AI 工具中自行修改
- **不做多语言提示词** — 统一中文，覆盖主要用户群
- **不做提示词版本管理** — 随代码 git 管理即可
- **不做在线 AI 生成** — 提示词是预先准备好的，不需要实时调 AI API
