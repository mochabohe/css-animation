# .agent

## 项目概览
- 项目名：`css-animation-demo`
- 技术栈：`Vite 5` + 原生 `HTML/CSS/JS`
- 目标：展示常见 CSS 动画效果（加载、弹跳、渐变、翻转）
- 语言：中文界面（`lang="zh-CN"`）

## 目录与职责
- `index.html`：页面结构与演示卡片布局
- `src/styles.css`：全部样式与动画关键帧
- `src/main.js`：入口文件（当前仅引入样式）
- `vite.config.js`：构建配置（`base: './'`，适配 GitHub Pages）

## 运行与构建
> 注意：请在 `css-animation-demo` 目录执行命令。

```bash
npm install
npm run dev
npm run build
npm run preview
```

## 开发约定
- 优先保持“纯 CSS 动画”方案，不引入不必要依赖。
- 新增动画时，按现有 `card` 模式扩展：HTML 一张卡片 + CSS 一组样式/关键帧。
- 保留无障碍属性（如 `aria-label`、`aria-hidden`）。
- 保留 `@media (prefers-reduced-motion: reduce)` 降级支持。
- 视觉风格遵循深色主题变量（`:root` 中的色板）。

## 代码风格
- 文件保持简洁：结构在 `index.html`，动画逻辑集中在 `src/styles.css`。
- 命名使用语义化 class（如 `loader`、`flip-card`、`ball-stage`）。
- 新增 keyframes 时按功能命名并与组件就近组织。

## 常见任务模板
### 新增动画卡片
1. 在 `index.html` 的 `.grid` 中添加 `article.card`。
2. 在 `src/styles.css` 中添加对应样式与 `@keyframes`。
3. 若涉及动效可用性，补充 `prefers-reduced-motion` 处理。

### 调整部署基路径
- 当前 `vite.config.js` 使用 `base: './'`，适合静态托管子路径部署。
- 若改为根路径部署，可改成 `base: '/'` 并验证资源引用。

## 禁止事项
- 不要删除现有动画演示卡片（除非明确需求）。
- 不要把动画逻辑拆散到多个随机文件，避免维护成本上升。
- 不要移除无障碍和 reduced-motion 支持。

## 交付检查清单
- 页面在 `npm run dev` 可正常打开。
- 新增/修改动画无明显布局回归。
- `npm run build` 通过。
- 动画在 reduced-motion 场景可降级。
